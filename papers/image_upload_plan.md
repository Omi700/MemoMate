# 手机或网页上传图片更换屏幕照片方案

## 目标

让设备左下角照片可以通过手机或网页更换，不需要每次重新编译固件。整体目标是低预算、低维护成本、两个人使用即可，不追求复杂后台。

当前设备是 ESP32-S3-RLCD-4.2，屏幕是黑白/反射式显示风格，最适合保存和显示 1-bit 黑白位图。图片上传后需要经过裁切、缩放、灰度增强、黑白抖动、圆角处理，再下发给设备。

## 推荐路线

最推荐先做“网页上传 + 设备局域网接收”的版本。

原因：

- 不需要服务器，成本为 0。
- 不需要微信小程序审核。
- 只要手机和设备连同一个 Wi-Fi，就可以上传图片。
- 实现难度低，适合当前阶段快速做出完整闭环。

核心流程：

```text
手机浏览器
  -> 打开设备局域网网页
  -> 上传图片
  -> ESP32 接收图片
  -> ESP32 或网页端转换为 1-bit 位图
  -> 保存到 Flash
  -> 屏幕刷新显示新照片
```

## 方案一：设备自带网页上传

ESP32 连接家里 Wi-Fi 后，启动一个本地 HTTP Server。手机访问设备 IP，例如：

```text
http://192.168.1.23
```

网页里提供一个图片上传按钮。用户选择照片后，网页把图片发给 ESP32。

### 优点

- 零服务器成本。
- 不依赖外部云平台。
- 家里网络可用时就能使用。
- 隐私更好，照片不上传到第三方服务器。

### 缺点

- 只能在同一个 Wi-Fi 下使用。
- 第一次需要知道设备 IP。
- ESP32 端处理 JPEG/PNG 会稍微麻烦。

### 低难度实现方式

不要让 ESP32 处理原始大照片。更好的做法是：在网页端用 Canvas 先完成图片处理。

网页端处理内容：

1. 读取用户选择的照片。
2. 裁切成左下框比例。
3. 缩放到固定位图尺寸，例如 `134 x 125`。
4. 转灰度。
5. 提高对比度。
6. 做 Floyd-Steinberg 黑白抖动。
7. 加圆角遮罩。
8. 打包成 XBM 风格的字节数组。
9. 通过 HTTP POST 发给 ESP32。

ESP32 只需要：

1. 接收已经处理好的 1-bit 位图数据。
2. 校验尺寸和数据长度。
3. 保存到 NVS 或 LittleFS。
4. UI 绘制时从 Flash 读取并调用 `u8g2_DrawXBMP()`。

这样 ESP32 的压力最小，也最省开发时间。

## 方案二：电脑/手机网页工具生成，再手动上传

做一个本地 HTML 工具，例如：

```text
tools/image-converter.html
```

这个网页不需要服务器，直接浏览器打开。它负责把照片转换成 `cat_photo.h` 或二进制 `.bin`。

### 优点

- 最简单。
- 完全零成本。
- 适合开发阶段反复调效果。
- 不需要 ESP32 先实现网页上传。

### 缺点

- 仍然需要电脑参与。
- 如果生成 `.h` 文件，还是要重新编译固件。
- 如果生成 `.bin`，还需要额外做一个上传入口。

### 适合阶段

适合作为下一步过渡。先把图片转换效果做好，再考虑远程或手机上传。

## 方案三：远程上传图片

如果以后你在学校，她在另一个地方，希望你远程给设备换照片，就需要一个公网中转。

最低预算推荐使用免费云服务：

- GitHub Pages：放一个图片上传网页。
- Cloudflare Workers：做一个轻量接口。
- Cloudflare R2 / Supabase Storage：存图片或转换后的位图。
- ESP32 定时请求接口，发现新图片版本就下载。

核心流程：

```text
你手机上传图片
  -> Cloudflare Worker 接收
  -> 保存转换后的 1-bit 位图
  -> 设备定时检查版本号
  -> 下载新位图
  -> 保存到 Flash
  -> 刷新屏幕
```

### 优点

- 可以远程更新。
- 你不需要和设备在同一个 Wi-Fi。
- 后续可以顺带扩展远程提醒、留言、配置同步。

### 缺点

- 需要一点云端开发。
- 需要处理简单鉴权。
- ESP32 需要联网下载。

### 费用

两个人使用、图片很小的情况下，理论上可以长期保持 0 元。

因为最终下发给设备的不是原图，而是 `134 x 125` 的 1-bit 位图。按行补齐到整字节后：

```text
ceil(134 / 8) * 125 = 17 * 125 = 2125 字节
```

一张图片大约 2 KB。即使每天更新很多次，流量也很小。

## 数据格式建议

不要把 JPEG 原图直接存到设备里。建议设备只保存转换后的位图。

推荐格式：

```text
Header:
  magic: "MORAIMG1"
  width:  uint16
  height: uint16
  length: uint32
  crc32:  uint32

Payload:
  1-bit XBM bitmap bytes
```

当前左下角照片建议尺寸：

```text
width  = 134
height = 125
bytes  = ((134 + 7) / 8) * 125 = 2125 bytes
```

使用固定尺寸的好处：

- 绘制代码简单。
- 不需要动态布局。
- 不容易越界。
- 后续换图片不会影响 UI。

## 设备端存储建议

优先用 LittleFS 或 SPIFFS 保存图片文件。

建议路径：

```text
/photo.bin
```

设备启动时：

1. 检查 `/photo.bin` 是否存在。
2. 如果存在，读取并显示用户上传图片。
3. 如果不存在，显示固件内置默认图片。

这样即使上传失败，设备也不会黑屏。

## 固件端需要改的内容

### 1. 增加文件系统

在 ESP-IDF 里启用 SPIFFS 或 LittleFS。低复杂度可以先用 SPIFFS。

需要内容：

- 分区表增加 `spiffs` 分区。
- 启动时挂载文件系统。
- 提供读取 `/photo.bin` 的函数。

### 2. 增加图片缓存

当前图片是编译进固件的：

```cpp
u8g2_DrawXBMP(u8g2, photo_x, photo_y, kCatPhotoWidth, kCatPhotoHeight, cat_photo_bits);
```

后续可以改成：

```cpp
if (LoadPhotoFromFlash(photo_buffer)) {
    u8g2_DrawXBMP(u8g2, photo_x, photo_y, kPhotoWidth, kPhotoHeight, photo_buffer);
} else {
    u8g2_DrawXBMP(u8g2, photo_x, photo_y, kCatPhotoWidth, kCatPhotoHeight, cat_photo_bits);
}
```

### 3. 增加 HTTP 上传接口

设备启动 HTTP Server，提供：

```text
GET  /          返回上传网页
POST /api/photo 接收 1-bit 位图数据
```

`POST /api/photo` 做这些检查：

- 图片宽度是否等于 134。
- 图片高度是否等于 125。
- payload 长度是否正确。
- CRC 是否正确。

检查通过后写入 `/photo.bin`，然后刷新 UI。

## 网页端需要实现的内容

网页可以非常简单：

- 选择图片按钮。
- 裁切预览区域。
- 黑白预览区域。
- 上传按钮。

关键处理逻辑：

1. 用 `<input type="file">` 读取图片。
2. 用 Canvas 绘制到 `134 x 125`。
3. 灰度化。
4. 对比度增强。
5. 黑白抖动。
6. 圆角外区域填白。
7. 每 8 个像素打包成一个字节。
8. POST 到 `/api/photo`。

这种方式对 ESP32 最友好，因为设备端不需要 JPEG 解码。

## 安全与隐私

局域网方案最低限度也建议加一个简单密码。

例如第一次访问网页时输入：

```text
upload password
```

设备端只接受带 token 的上传请求：

```text
POST /api/photo?token=xxxx
```

如果只是两个人使用，先用固定密码即可。后续远程版再考虑更规范的登录。

## 推荐实施步骤

### 第一阶段：保持手动转换

目标：把图片显示效果调到满意。

要做：

- 保留现在的 `img1.jpg -> cat_photo.h` 转换脚本。
- 多试几张照片，确定最佳裁切比例、对比度和抖动参数。
- 固定最终位图尺寸。

### 第二阶段：做本地网页转换工具

目标：不用 Python，也能在浏览器里预览黑白效果。

要做：

- 新建 `tools/image-converter.html`。
- 浏览器中选择照片。
- 实时显示黑白预览。
- 导出 `.h` 或 `.bin`。

### 第三阶段：设备局域网上传

目标：手机连同一个 Wi-Fi，就能直接换照片。

要做：

- ESP32 接入 Wi-Fi。
- ESP32 启动 HTTP Server。
- 网页端完成图片转换。
- 上传二进制位图到设备。
- 设备保存到 SPIFFS 并刷新屏幕。

### 第四阶段：远程上传

目标：不在同一个 Wi-Fi，也能换照片。

要做：

- Cloudflare Worker 或 Supabase 免费服务。
- 上传网页部署到 GitHub Pages。
- ESP32 定时检查新图片版本。
- 下载并保存新图片。

## 当前最建议的下一步

先做第二阶段：`tools/image-converter.html`。

原因：

- 不需要动设备网络。
- 不需要服务器。
- 可以快速比较不同照片的黑白效果。
- 后面第三阶段的网页端代码可以直接复用。

等转换效果稳定，再接 ESP32 的 HTTP 上传接口。这样开发风险最低，也最省钱。
